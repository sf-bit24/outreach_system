, il existe des alternatives open source ou des wrappers pour d'autres services gratuits ou freemium.

*   **`resend/resend-python` [1] :** Le SDK officiel Python pour Resend. Facile à intégrer pour l'envoi d'emails transactionnels. La version gratuite permet un certain volume d'envois.
*   **`mailgun/mailgun-python` [2] :** SDK Python pour Mailgun, qui offre également un plan gratuit avec un volume d'emails limité. Peut être une alternative ou un fallback.
*   **`sendgrid/sendgrid-python` [3] :** SDK Python pour SendGrid, avec un plan gratuit similaire.

### 4.2. Gestion de File d'Attente et Tâches Asynchrones

Pour gérer l'envoi espacé et les tâches en arrière-plan, un système de file d'attente est essentiel.

*   **`celery/celery` [4] :** Un système de file d'attente de tâches distribuées robuste et mature pour Python. Il permet de planifier et d'exécuter des tâches d'envoi d'emails de manière asynchrone et espacée.
*   **`rq/rq` [5] :** Une alternative plus simple à Celery, basée sur Redis. Idéale pour des projets de taille moyenne.
*   **`apscheduler/APScheduler` [6] :** Une bibliothèque Python pour planifier des tâches (jobs) en mémoire, sur disque ou via une base de données. Utile pour des tâches récurrentes ou des envois à des moments précis.

### 4.3. Suivi des Réponses et Webhooks

Pour détecter les réponses, il faut configurer des webhooks avec le service d'envoi d'emails et un petit serveur HTTP.

*   **`flask/flask` [7] ou `tiangolo/fastapi` [8] :** Des frameworks web légers pour créer un endpoint HTTP qui recevra les notifications de réponses (webhooks) du service d'envoi d'emails. Ce endpoint mettra à jour le statut du lead dans la base de données.

## 5. Flux de Travail de l'Orchestrateur d'Envoi

1.  **Chargement des Messages :** Le module charge les messages personnalisés du fichier `data/processed/personalized_messages.csv`.
2.  **Planification de l'Envoi :** Pour chaque message, une tâche d'envoi est ajoutée à la file d'attente (ex: Celery/RQ) avec un délai aléatoire (configuré via `.env`).
3.  **Exécution de la Tâche d'Envoi :**
    *   La tâche récupère le message et les informations du lead.
    *   Elle utilise l'API Resend (ou autre) pour envoyer l'email.
    *   Elle s'assure que l'email contient l'identification de l'expéditeur et le lien de désabonnement.
    *   Le statut de l'envoi est enregistré dans la base de données (ex: `sent`, `failed`).
4.  **Gestion des Webhooks :**
    *   Un petit serveur Flask/FastAPI écoute les webhooks du service d'envoi.
    *   Lorsqu'une réponse ou un désabonnement est détecté, le statut du lead est mis à jour dans la base de données.
    *   Les désabonnements sont traités immédiatement (le contact est marqué comme `unsubscribed` et ne recevra plus d'emails).
5.  **Journalisation :** Toutes les actions sont journalisées pour des raisons de traçabilité et de conformité.

## 6. Conformité LCAP (Rappel)

Le module d'envoi doit impérativement garantir :

*   **Consentement Tacite :** Confirmer que le lead a été validé par le module d'enrichissement comme étant conforme à l'exemption de "publication bien en vue".
*   **Identification Claire :** Chaque email doit clairement indiquer qui l'envoie (nom de l'entreprise, adresse physique ou email/téléphone de contact).
*   **Mécanisme de Désabonnement :** Un lien de désabonnement facile à trouver et fonctionnel doit être inclus dans chaque email. La demande doit être traitée dans les 10 jours ouvrables.

from resend import Resend

class EmailSender:
    def __init__(self):
        self.resend = Resend(api_key=os.getenv("RESEND_API_KEY"))
        self.sender_email = os.getenv("SENDER_EMAIL")
        self.sender_name = os.getenv("SENDER_NAME")

    def send_email(self, recipient_email, subject, body):
        try:
            # Assurez-vous que le corps de l'email inclut l'identification et le lien de désabonnement
            full_body = f"{body}\n\n---\n{self.sender_name}\n{self.sender_email}\nPour vous désabonner: [LIEN_DESABONNEMENT]"

            self.resend.emails.send({
                "from": f"{self.sender_name} <{self.sender_email}>",
                "to": [recipient_email],
                "subject": subject,
                "html": full_body,
            })
            return {"status": "success"}
        except Exception as e:
            return {"status": "failed", "error": str(e)}
