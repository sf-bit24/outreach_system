# Système d'Outreach Intelligent - Pipeline Backend

Ce projet est conçu pour automatiser la prospection B2B de manière éthique et personnalisée, en mettant l'accent sur la qualité de la recherche client plutôt que sur le volume d'envoi.

## 1. Architecture du Pipeline

Le backend est divisé en 4 étapes clés :

### Étape 1 : Acquisition de Données (Scrapers)
- **Source REQ :** Importation des données du Registre des Entreprises du Québec.
- **Source Apollo/LinkedIn :** Extraction ciblée des décideurs (ICP).
- **Validation :** Vérification immédiate de la "publication manifeste" de l'email (conformité LCAP).

### Étape 2 : Recherche et Enrichissement (Intelligence)
- **Web Scraping :** Analyse du site web de la cible pour comprendre son offre.
- **Signal d'Intention :** Vérification si l'entreprise recrute (Job Boards).
- **Validation Email :** Test de délivrabilité (SMTP check) pour protéger la réputation du domaine.

### Étape 3 : Moteur de Personnalisation (AI Engine)
- **Analyse Contextuelle :** Utilisation de l'IA pour synthétiser les données collectées.
- **Génération d'Accroche :** Création d'une phrase d'introduction unique basée sur un fait réel (ex: "J'ai vu que vous recrutiez un nouveau VP Sales...").
- **Stratégie POC :** Insertion dynamique de la preuve que le système a fonctionné pour les atteindre.

### Étape 4 : Orchestration de l'Envoi (Sender)
- **Gestion des Files d'Attente :** Envoi espacé pour simuler un comportement humain.
- **Suivi des Réponses :** Détection automatique des réponses pour stopper les séquences.

## 2. Structure des Dossiers

```text
src/
├── scrapers/      # Scripts d'acquisition (REQ, Apollo, Maps)
├── enrichment/    # Analyse de sites web et validation d'emails
├── ai_engine/     # Génération de contenu personnalisé via LLM
├── sender/        # Intégration Resend et gestion des séquences
└── utils/         # Helpers (logging, database, config)
data/
├── raw/           # Données brutes de scraping
└── processed/     # Leads qualifiés et enrichis
```

## 3. Installation Rapide

1. Clonez le repo.
2. Copiez `.env.example` en `.env` et remplissez vos accès.
3. Installez les dépendances : `pip install -r requirements.txt`.
